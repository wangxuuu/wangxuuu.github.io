Contrastive Learning

Notes on Contrastive Learning

In recently years, numerous self-supervised learning methods have been proposed for learning representations in image or natural language. However, their performance is still below the supervised counterparts.

Things change when some frameworks proposed based on contrastive learning, such as SimCLR and MoCo. These approaches not only improves upon the previous SOTA self-supervised learning methods but also beat some supervised learning method on ImageNet classification.

This approach is generated from noise contrastive learning (NCE), which aims to avoid the partition function by introducing a classfier to distingush the noise and the raw data samples. Recently, many research works leaveage contrastive learning in NLP to reduce the computation cost in softmax mapping.

1.1 Main Idea of Contrastive Learning

The core idea of contrastive learning is to learn a function modeled by a deep network(encoder/representation) that maps semantically nearby points (positive pairs) closer together in the embedding space, while pushing apart points that are dissimilar (nagative pairs). It attempts to teach machines to distinguish between similar and dissimilar things.

Supervised contrastive learning vs. self-supervised learning

  1. In supervised contrastive learning, the labels are provided and we take the images from the same class as positive and map them to nearby points in a low-dimensional hyperspace (encoding space).
  2. In self-supervised contrastive learning, the lables are not provided. Hence normally we generate the data augmentations of a given sample as positives and the negatives are randomly sampled from the mini-batch.

Problems

  1. In self-supervised cases, only a single positive is used. WHY? Can we select more positives?
  2. With the labels in supervised learning, we can also select multiple positive pairs per anchor in different classes. Is there any benefit?

1.3 Self-supervised Contrastive Learning

via Ankesh Anand

Contrastive methods learn representations by contrasting positive and negative examples. Although not a new paradigm, they have led to great empirical success in computer vision tasks with unsupervised contrastive pre-training.

  • (unsupervised learning) Contrastive methods trained on unlabelled ImageNet data and evaluated with a linear classifier now surpass the accuracy of supervised AlexNet.

  • (supervised learning) They also exhibit significant data efficiency when learning from labelled data compared to purely supervised learning (Data-Efficient CPC, Hénaff et al., 2019).

  • (Downstream tasks) Contrastive pre-training on ImageNet successfully transfers to other downstream tasks and outperforms the supervised pre-training counterparts.

    Generally, contrastive learning focus on abstract latent factors and high-level features rather than pixel-level details in VAE or GAN. Pixel-based objectives often assume independence between each pixel, thereby reducing their ability to model correlations or complex structure.

Contrastive Predictive Coding

Contrastive Predictive Coding (CPC, van den Oord et al., 2018) is a contrastive method that can be applied to any form of data that can be expressed in an ordered sequence: text, speech, video, even images (an image can be seen as a sequence of pixels or patches).

where the contrastive task in CPC is set as follows. Let $\{x_1,\dots,x_N\}$ be a sequence of data points, and $x_t$ be an anchor data point. Then,

  • $x_{t+k}$ will be a positive sample for this anchor.
  • A data point $x_{t^*}$ randomly sampled from the sequence will be a negative sample.
